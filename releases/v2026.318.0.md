# v2026.318.0

> 发布日期：2026-03-18

## 亮点

- **插件框架与 SDK** — 完整的插件系统，包含运行时生命周期管理、CLI 工具、设置 UI、面包屑与插槽扩展、领域事件桥接，以及一个功能齐全的示例。Plugin SDK 现在包含文档 CRUD 方法和测试工具。([#904](https://github.com/paperclipai/paperclip/pull/904), [#910](https://github.com/paperclipai/paperclip/pull/910), [#912](https://github.com/paperclipai/paperclip/pull/912), [#909](https://github.com/paperclipai/paperclip/pull/909), [#1074](https://github.com/paperclipai/paperclip/pull/1074), @gsxdsm, @mvanhorn, @residentagent)
- **升级的成本与预算管理** — 改进了成本追踪与预算管理界面。([#949](https://github.com/paperclipai/paperclip/pull/949))
- **事务文档与附件** — 事务现在支持内联文档编辑、创建前的文件暂存、文档深链接、复制与下载操作，以及实时事件刷新。([#899](https://github.com/paperclipai/paperclip/pull/899))
- **Hermes 智能体适配器** — 新的 `hermes_local` 适配器，支持将 Hermes CLI 作为智能体后端。([#587](https://github.com/paperclipai/paperclip/pull/587), @teknium1)
- **执行工作区（实验性）** — 为智能体运行提供隔离的执行工作区，包括工作区操作追踪、可复用工作区去重以及工作产物管理。项目级工作区策略可配置。([#1038](https://github.com/paperclipai/paperclip/pull/1038))
- **心跳令牌优化** — 心跳周期现在会跳过冗余的令牌消耗。

## 改进

- **会话压缩感知适配器** — 压缩逻辑现在会尊重每个适配器的上下文限制。
- **公司 Logo** — 上传并展示公司 Logo，包含 SVG 清洗和增强的资产响应安全头。([#162](https://github.com/paperclipai/paperclip/pull/162), @JonCSykes)
- **应用版本标签** — 侧边栏现在显示当前运行的 Paperclip 版本。([#1096](https://github.com/paperclipai/paperclip/pull/1096), @saishankar404)
- **项目标签页缓存** — 活跃的项目标签页会按项目记住；标签页已重命名并重新排序。([#990](https://github.com/paperclipai/paperclip/pull/990))
- **事务复制到剪贴板** — 事务详情头部现在包含复制按钮；HTML 实体不再泄漏到复制的文本中。([#990](https://github.com/paperclipai/paperclip/pull/990))
- **"我"与"未指派"经办人选项** — 快速筛选当前用户和未指派事务的经办人选项。([#990](https://github.com/paperclipai/paperclip/pull/990))
- **新建事务对话框跳过预填字段** — 当指派人和项目字段已预填时，Tab 键顺序现在会跳过它们。([#990](https://github.com/paperclipai/paperclip/pull/990))
- **Worktree 清理命令** — 新的 `worktree:cleanup` 命令、环境变量默认值以及 worktree 分支的自动前缀。([#1038](https://github.com/paperclipai/paperclip/pull/1038))
- **发布自动化** — 自动化的 canary 和 stable 发布工作流，支持 npm 可信发布和来源元数据。([#1151](https://github.com/paperclipai/paperclip/pull/1151), [#1162](https://github.com/paperclipai/paperclip/pull/1162))
- **文档链接** — 侧边栏文档链接现在指向外部 docs.paperclip.ing。
- **新用户引导初始任务延迟** — 初始任务现在在用户启动前不再创建。

## 修复

- **嵌入式 PostgreSQL 加固** — 启动适配、数据目录验证和 UTF-8 编码现在都能可靠处理。(@vkartaviy)
- **`os.userInfo()` 防护** — 仅含 UID 用户的容器不再崩溃；HOME 已从缓存键中排除。([#1145](https://github.com/paperclipai/paperclip/pull/1145), @wesseljt)
- **opencode-local HOME 解析** — 模型发现现在使用 `os.userInfo()` 而非依赖 HOME 环境变量。([#1145](https://github.com/paperclipai/paperclip/pull/1145), @wesseljt)
- **dotenv cwd 回退** — 当 `.paperclip/.env` 缺失时，服务器现在从当前工作目录加载 `.env`。([#834](https://github.com/paperclipai/paperclip/pull/834), @mvanhorn)
- **插件事件订阅接线** — 修复了订阅清理、过滤器可空性和过期图表。([#988](https://github.com/paperclipai/paperclip/pull/988), @leeknowsai)
- **插件插槽渲染** — 修正了插件 UI 扩展的插槽注册和渲染。([#916](https://github.com/paperclipai/paperclip/pull/916), [#918](https://github.com/paperclipai/paperclip/pull/918), @gsxdsm)
- **归档项目用户体验** — 归档操作现在会导航到仪表板并显示提示；用内联确认替代了 `window.confirm`。
- **Markdown 编辑器间距** — 图片拖放/粘贴添加正确的换行；标题上边距增大。
- **工作区表单刷新** — 通过 URL key 访问项目时表单现在会刷新，并允许空保存。
- **遗留迁移对账** — 修复了现有安装的迁移对账问题。
- **`archivedAt` 类型强制转换** — 在 Drizzle 更新前进行字符串到日期的转换，防止类型错误。
- **智能体 HOME 环境变量** — 子智能体进程的 `AGENT_HOME` 现在设置正确。([#864](https://github.com/paperclipai/paperclip/pull/864))
- **侧边栏滚动条悬停轨道** — 修复了悬停时滚动条轨道的可见性。([#919](https://github.com/paperclipai/paperclip/pull/919))
- **非配置标签页上的粘性保存栏** — 已隐藏以防止布局推挤。
- **空目标显示** — 移除了空目标中的"None"文字。
- **运行页面内边距** — 移除了不必要的右侧内边距。
- **Codex 启动日志** — 现在作为标准输出而非标准错误处理。
- **开发运行器语法** — 修复了插件开发运行器中的语法问题。([#914](https://github.com/paperclipai/paperclip/pull/914), @gsxdsm)
- **进程列表** — 修复了进程列表渲染。([#903](https://github.com/paperclipai/paperclip/pull/903), @gsxdsm)

## 升级指南

启动时将自动运行十个新的数据库迁移（`0028`–`0037`）：

- **迁移 0028–0029** 添加插件框架相关表。
- **迁移 0030–0037** 扩展模式以支持事务文档、执行工作区、公司 Logo、成本追踪和插件增强。

所有迁移均为增量操作（新表和新列）— 不修改现有数据。标准的 `paperclipai` 启动将自动应用它们。

如果您使用 `.env` 文件，请注意服务器现在会在找不到 `.paperclip/.env` 时回退到从当前工作目录加载 `.env`。
