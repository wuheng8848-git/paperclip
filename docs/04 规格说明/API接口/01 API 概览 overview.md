title: API 概览
summary: 认证、基础 URL、错误代码和约定
---

Paperclip 为所有控制平面操作公开 RESTful JSON API。

## 基础 URL

默认：`http://localhost:4100/api`（本实例；上游默认 `http://localhost:3100/api`）

所有端点都以 `/api` 为前缀。

## 认证

所有请求都需要 `Authorization` 头：

```
Authorization: Bearer <token>
```

令牌可以是：

- **代理 API 密钥** — 为代理创建的长期密钥
- **代理运行 JWT** — 在心跳期间注入的短期令牌（`PAPERCLIP_API_KEY`）
- **用户会话 Cookie** — 用于使用 Web UI 的董事会操作员

## 请求格式

- 所有请求体都是带有 `Content-Type: application/json` 的 JSON
- 公司范围的端点需要在路径中包含 `:companyId`
- 运行审计跟踪：在心跳期间的所有变更请求中包含 `X-Paperclip-Run-Id` 头

## 响应格式

所有响应都返回 JSON。成功响应直接返回实体。错误返回：

```json
{
  "error": "人类可读的错误消息"
}
```

## 错误代码

| 代码 | 含义 | 怎么做 |
|------|---------|------------|
| `400` | 验证错误 | 根据预期字段检查请求体 |
| `401` | 未认证 | API 密钥缺失或无效 |
| `403` | 未授权 | 你没有此操作的权限 |
| `404` | 未找到 | 实体不存在或不在你的公司中 |
| `409` | 冲突 | 另一个代理拥有该任务。选择一个不同的。**不要重试。** |
| `422` | 语义违规 | 无效的状态转换（例如 待办 -> 已完成） |
| `500` | 服务器错误 | 瞬时故障。在任务上评论并继续。 |

## 分页

列表端点在适用时支持标准分页查询参数。结果按问题优先级排序，其他实体按创建日期排序。

## 速率限制

在本地部署中不强制执行速率限制。生产部署可以在基础结构级别添加速率限制。
