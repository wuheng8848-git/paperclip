title: 批准
summary: 雇佣和策略的治理流程
---

Paperclip 包括批准门，让人类董事会操作员控制关键决策。

## 批准类型

### 雇佣代理

当代理（通常是经理或 CEO）想要雇佣新的下属时，他们会提交雇佣请求。这将创建一个 `hire_agent` 批准，显示在你的批准队列中。

批准包括提议的代理名称、角色、能力、适配器配置和预算。

### CEO 策略

CEO 的初始战略计划需要董事会批准，然后 CEO 才能开始将任务移至 `in_progress`。这确保了对公司方向的人类签字。

## 批准工作流

```
待处理 -> 已批准
        -> 已拒绝
        -> 需要修订 -> 重新提交 -> 待处理
```

1. 代理创建批准请求
2. 它出现在你的批准队列中（UI 中的批准页面）
3. 你审查请求详细信息和任何链接的问题
4. 你可以：
   - **批准** — 操作继续进行
   - **拒绝** — 操作被拒绝
   - **请求修订** — 要求代理修改并重新提交

## 审查批准

从批准页面，你可以看到所有待处理的批准。每个批准显示：

- 谁请求的以及为什么
- 链接的问题（请求的上下文）
- 完整的有效负载（例如，雇佣的提议代理配置）

## 董事会覆盖权力

作为董事会操作员，你还可以：

- 随时暂停或恢复任何代理
- 终止任何代理（不可逆转）
- 将任何任务重新分配给不同的代理
- 覆盖预算限制
- 直接创建代理（绕过批准流程）
