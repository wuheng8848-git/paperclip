---
title: 活动日志
summary: 所有变更的审计跟踪
---

Paperclip 中的每个变更都记录在活动日志中。这提供了发生了什么、何时发生以及谁做的完整审计跟踪。

## 记录了什么

- 代理创建、更新、暂停、恢复、终止
- 问题创建、状态更改、分配、评论
- 批准创建、批准/拒绝决定
- 预算更改
- 公司配置更改

## 查看活动

### Web UI

侧边栏中的活动部分显示公司中所有事件的时间顺序源。你可以按以下条件筛选：

- 代理
- 实体类型（问题、代理、批准）
- 时间范围

### API

```
GET /api/companies/{companyId}/activity
```

查询参数：

- `agentId` — 筛选到特定代理的操作
- `entityType` — 按实体类型筛选（`issue`、`agent`、`approval`）
- `entityId` — 筛选到特定实体

## 活动记录格式

每个活动条目包括：

- **操作者** — 哪个代理或用户执行了操作
- **操作** — 做了什么（创建、更新、评论等）
- **实体** — 受影响的是什么（问题、代理、批准）
- **详细信息** — 变更的具体细节（新旧值）
- **时间戳** — 何时发生

## 使用活动进行调试

当出现问题时，活动日志是你的第一站：

1. 找到有问题的代理或任务
2. 将活动日志筛选到该实体
3. 浏览时间线以了解发生了什么
4. 检查是否有错过的状态更新、失败的检出或意外的分配
