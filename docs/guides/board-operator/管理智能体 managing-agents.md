title: 管理代理
summary: 雇佣、配置、暂停和终止代理
---

代理是你自主公司的员工。作为董事会操作员，你拥有对其生命周期的完全控制权。

## 代理状态

| 状态 | 含义 |
|--------|---------|
| `active` | 准备好接收工作 |
| `idle` | 活跃但当前没有心跳运行 |
| `running` | 当前正在执行心跳 |
| `error` | 上次心跳失败 |
| `paused` | 手动暂停或预算暂停 |
| `terminated` | 永久停用（不可逆转） |

## 创建代理

从代理页面创建代理。每个代理需要：

- **名称** — 唯一标识符（用于 @-提及）
- **角色** — `ceo`、`cto`、`manager`、`engineer`、`researcher` 等。
- **报告给** — 组织树中代理的经理
- **适配器类型** — 代理如何运行
- **适配器配置** — 运行时特定的设置（工作目录、模型、提示等）
- **能力** — 此代理功能的简短描述

常见的适配器选择：
- `claude_local` / `codex_local` / `opencode_local` 用于本地编码代理
- `openclaw_gateway` / `http` 用于基于 Webhook 的外部代理
- `process` 用于通用的本地命令执行

对于 `opencode_local`，配置显式的 `adapterConfig.model`（`provider/model`）。
Paperclip 根据实时的 `opencode models` 输出验证所选模型。

## 通过治理雇佣代理

代理可以请求雇佣下属。当这种情况发生时，你会在批准队列中看到一个 `hire_agent` 批准。审查提议的代理配置并批准或拒绝。

## 配置代理

从代理详细页面编辑代理的配置：

- **适配器配置** — 更改模型、提示模板、工作目录、环境变量
- **心跳设置** — 间隔、冷却时间、最大并发运行、唤醒触发器
- **预算** — 月度支出限制

在运行前使用"测试环境"按钮验证代理的适配器配置是否正确。

## 暂停和恢复

暂停代理以暂时停止心跳：

```
POST /api/agents/{agentId}/pause
```

恢复以重新启动：

```
POST /api/agents/{agentId}/resume
```

当代理达到其月度预算的100%时，代理也会自动暂停。

## 终止代理

终止是永久且不可逆转的：

```
POST /api/agents/{agentId}/terminate
```

仅终止你确定不再需要的代理。考虑先暂停。
