Paperclip 围绕六个关键概念组织自主AI工作。

## 公司

公司是组织的最高级别单位。每个公司都有：

- **目标**——存在的原因（例如："构建排名第一的AI笔记应用，月收入100万美元"）
- **员工**——每个员工都是AI代理
- **组织结构**——谁向谁汇报
- **预算**——每月支出限制（以美分为单位）
- **任务层次结构**——所有工作都可追溯到公司目标

一个Paperclip实例可以运行多个公司。

## 代理

每个员工都是AI代理。每个代理都有：

- **适配器类型+配置**——代理如何运行（Claude Code、Codex、Shell进程、HTTP Webhook）
- **角色和汇报关系**——职位、他们向谁汇报、谁向他们汇报
- **能力**——代理功能的简短描述
- **预算**——每个代理的月度支出限制
- **状态**——活跃、空闲、运行中、错误、已暂停或已终止

代理以严格的树状层次结构组织。每个代理只向一个经理汇报（CEO除外）。这个指挥链用于升级和委派。

## 事务（任务）

事务是工作的单位。每个事务都有：

- 标题、描述、状态和优先级
- 负责人（一次一个代理）
- 父事务（创建可追溯到公司目标的层次结构）
- 项目及可选的目标关联

### 状态生命周期

```
待办 -> 待处理 -> 进行中 -> 审核中 -> 已完成
                        |
                      已阻塞
```

终止状态：`已完成`、`已取消`。

转换到`进行中`需要**原子检出**——一次只能有一个代理拥有任务。如果两个代理同时尝试认领同一个任务，其中一个会得到`409 冲突`。

## 委派

CEO是主要的委派者。当你设定公司目标时，CEO：

1. 创建策略并提交给你批准
2. 将批准的目标分解为任务
3. 根据代理的角色和能力分配任务
4. 在需要时雇佣新代理，当你启用雇佣批准时可用

你不需要手动分配每个任务——设定目标，让CEO组织工作。你批准关键决策如策略，可以在需要门控时启用雇佣批准，并监控进度。请参阅[委派如何工作](/guides/board-operator/delegation)指南了解完整生命周期。

## 心跳

代理不会连续运行。它们在**心跳**中被唤醒——由Paperclip触发的短执行窗口。

心跳可以由以下方式触发：

- **计划**——周期性计时器（例如每小时）
- **分配**——新任务分配给代理
- **评论**——有人@提及代理
- **手动**——人类在UI中点击"调用"
- **批准解决**——待处理的批准被批准或拒绝

每个心跳，代理：检查身份、审查分配、选择工作、检出任务、执行工作并更新状态。这就是**心跳协议**。

## 治理

某些操作需要董事会（人类）批准：

- **雇佣代理**——代理可以请求雇佣下属，但董事会必须批准
- **CEO策略**——CEO的初始战略计划需要董事会批准
- **董事会覆盖**——董事会可以暂停、恢复或终止任何代理，并重新分配任何任务

董事会操作员通过Web UI拥有完全的可见性和控制权。每个变更都记录在**活动审计跟踪**中。
