# 派单与能力表

**定位：** 现阶段 **谁派、派谁、叫什么名、母单/子单怎么分**——操作真源。终局「一句话全自动」**不是** done 标准（见 §6）。

董事会派单全文：[`参考/董事会派单协议`](参考/董事会派单协议%20board-dispatch-protocol.md)

---

## 1. 三句话

1. **董事会**（人 + 对话框强脑）定边界、拆单、拍板 done。  
2. **Paperclip** 存事务、唤醒、run、时间线——**不**替董事会扩 scope。  
3. **执行 agent** 只做窄任务：**读 issue → 干 → 回写**，不自派 CEO 链。

---

## 2. 能力表 ≠ 三省六部

公司里的 agent 是 **模型 × CLI 壳 × 职能** 的工具箱，不是让 CEO/CTO 每天自动串公文。

| 误区 | 正解 |
| --- | --- |
| 组织树 = 自动派单链 | **董事会**派单；`reportsTo`  mostly 分组 |
| 看 adapter 类型猜人 | 看 **agent 名 + model** |
| 15 岗全开 heartbeat | **Pause + invoke**（见 [实例开关](实例开关与阶段零%20instance-switches.md)） |

**强带弱：** 探查 / 同质大批 → Flash（如 v4f）；分析 / 强改码 → Pro（如 ds4p）；裁决 → 人 + 强模型。

---

## 3. 命名（派单时看全名）

推荐格式（真源：`执行/005` §11.1）：

```text
{职能}-{壳子}-{模型}
```

例：`开发-CodeBuddy-glm5.1`、`快速探查-ds4f`

- **职能：** 前端 / 开发 / 测试 / CTO…  
- **壳子：** CodeBuddy / Qwen / Cursor  
- **模型：** glm5.1、qwen3.6plus、DP4P…  

**过渡短名**（`执行`、`质量`）易混 T5/T6——派单时优先用 **完整三段名**。

---

## 4. 长单怎么拆（T 链）

长任务按 **T1→…→T9** 分段，**每段**从能力表挑人，不是一人扛全链。

| 段 | 典型派谁 | 模型档 |
| --- | --- | --- |
| T2 探查 | 探查 / 快速探查 | **Flash** |
| T3 分析 | CTO / 分析师 | **Pro** |
| T5 执行 | 开发 / 前端 | 按任务 + 壳子 |
| T6 测试 | 测试岗 | QA 向 |

T 链详表：`长期需求/08 编排规范`

---

## 5. 派单四件套（现阶段契约）

API 逐步示例见 [派单API调用说明 dispatch-api-cookbook](派单API调用说明%20dispatch-api-cookbook.md)。

1. **事务边界** — 母单规划时 **无经办**；实现只派 **子单**（母单评论别误唤醒经办，见 009 §12）。  
2. **执行文档** — 子单 description 写清 **路径 + 完成定义**。  
3. **工人技能** — 收敛为执行 + 回写；不自带拆单。  
4. **结单证据** — 文档回写 + run + 董事会 **done**（见 [结单与回写](结单与回写%20completion-and-writeback.md)）。

```text
目标 → issue/子单 → invoke → run → 回写 → done
```

## 6. 现阶段 vs 终局

| | **现阶段（验收用这个）** | **终局（梦想）** |
| --- | --- | --- |
| 输入 | 目标 + 拆单 + 指派 | 一句话目标 |
| 流转 | issue / blocked / run | 全自动 |
| 验收 | 回写 + run + done | 模型自验 |

终局叙述：`长期需求/05` §7

## 7. 要压的噪音

| 噪音 | 手段 |
| --- | --- |
| 无活 timer / recovery | [实例开关](实例开关与阶段零%20instance-switches.md) |
