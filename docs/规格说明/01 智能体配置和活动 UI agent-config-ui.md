# 代理配置和活动 UI

## 上下文

代理是 Paperclip 公司的员工。每个代理都有一个适配器类型（`claude_local`、`codex_local`、`process`、`http`），决定了它如何运行，组织结构图中的位置（它向谁汇报），心跳策略（如何/何时唤醒），以及预算。`/agents` 处的 UI 需要支持创建和配置代理、查看其组织层次结构，以及检查它们一直在做什么——它们的运行历史、实时日志和累积成本。

本规范涵盖三个界面：

1. **代理创建对话框** — "新建代理"流程
2. **代理详细信息页面** — 配置、活动和日志
3. **代理列表页面** — 对现有列表的改进

---

## 1. 代理创建对话框

遵循现有的 `NewIssueDialog` / `NewProjectDialog` 模式：一个带有展开/最小化切换、公司徽标面包屑和 Cmd+Enter 提交的 `Dialog` 组件。

### 字段

**身份（始终可见）：**

| 字段 | 控件 | 必需 | 默认值 | 备注 |
|-------|---------|----------|---------|-------|
| 名称 | 文本输入（大，自动聚焦） | 是 | -- | 例如 "Alice"、"Build Bot" |
| 标题 | 文本输入（副标题样式） | 否 | -- | 例如 "VP of Engineering" |
| 角色 | 芯片弹出窗口（选择） | 否 | `general` | 值来自 `AGENT_ROLES`：ceo、cto、cmo、cfo、engineer、designer、pm、qa、devops、researcher、general |
| 报告给 | 芯片弹出窗口（代理选择） | 否 | -- | 公司中现有代理的下拉列表。如果这是第一个代理，自动将角色设置为 `ceo` 并将报告给变灰。否则，除非角色是 `ceo`，否则必需。 |
| 能力 | 文本输入 | 否 | -- | 此代理功能的自由文本描述 |

**适配器（可折叠部分，默认展开）：**

| 字段 | 控件 | 默认值 | 备注 |
|-------|---------|---------|-------|
| 适配器类型 | 芯片弹出窗口（选择） | `claude_local` | `claude_local`、`codex_local`、`process`、`http` |
| 测试环境 | 按钮 | -- | 运行适配器特定的诊断，并为当前未保存的配置返回通过/警告/失败检查 |
| CWD | 文本输入 | -- | 本地适配器的工作目录 |
| 提示模板 | 文本区域 | -- | 支持 `{{ agent.id }}`、`{{ agent.name }}` 等 |
| 模型 | 文本输入 | -- | 可选模型覆盖 |

**适配器特定字段（根据适配器类型显示/隐藏）：**

*claude_local:*
| 字段 | 控件 | 默认值 |
|-------|---------|---------|
| 每运行最大轮次 | 数字输入 | 80 |
| 跳过权限 | 切换 | true |

*codex_local:*
| 字段 | 控件 | 默认值 |
|-------|---------|---------|
| 搜索 | 切换 | false |
| 绕过沙箱 | 切换 | true |

*process:*
| 字段 | 控件 | 默认值 |
|-------|---------|---------|
| 命令 | 文本输入 | -- |
| 参数 | 文本输入（逗号分隔） | -- |

*http:*
| 字段 | 控件 | 默认值 |
|-------|---------|---------|
| URL | 文本输入 | -- |
| 方法 | 选择 | POST |
| 头 | 键值对 | -- |

**运行时（可折叠部分，默认折叠）：**

| 字段 | 控件 | 默认值 |
|-------|---------|---------|
| 上下文模式 | 芯片弹出窗口 | `thin` |
| 月度预算（美分） | 数字输入 | 0 |
| 超时（秒） | 数字输入 | 900 |
| 宽限期（秒） | 数字输入 | 15 |
| 额外参数 | 文本输入 | -- |
| 环境变量 | 键值对编辑器 | -- |

**心跳策略（可折叠部分，默认折叠）：**

| 字段 | 控件 | 默认值 |
|-------|---------|---------|
| 启用 | 切换 | true |
| 间隔（秒） | 数字输入 | 300 |
| 分配时唤醒 | 切换 | true |
| 按需唤醒 | 切换 | true |
| 自动化唤醒 | 切换 | true |
| 冷却时间（秒） | 数字输入 | 10 |

### 行为

- 提交时，调用 `agentsApi.create(companyId, data)`，其中 `data` 将身份字段打包在顶层，适配器特定字段打包到 `adapterConfig` 中，心跳/运行时打包到 `runtimeConfig` 中。
- 创建后，导航到新代理的详细信息页面。
- 如果公司有零个代理，则预填充角色为 `ceo` 并禁用报告给。
- 当适配器类型更改时，适配器配置部分会更新其可见字段，保留任何共享字段值（cwd、promptTemplate 等）。

## 2. 代理详细信息页面

重构现有的选项卡布局。保留标题（名称、角色、标题、状态徽标、操作按钮）并添加更丰富的选项卡。

### 标题

```
[StatusBadge]  代理名称                    [Invoke] [Pause/Resume] [...]
               角色 / 标题
```

`[...]` 溢出菜单包含：终止、重置会话、创建 API 密钥。

### 选项卡

#### 概述选项卡

两栏布局：左栏是摘要卡片，右栏是组织位置。

**摘要卡片：**
- 适配器类型 + 模型（如果设置）
- 心跳间隔（例如"每5分钟"）或"已禁用"
- 上次心跳时间（相对时间，例如"3分钟前"）
- 会话状态："活动（会话 abc123...）"或"无会话"
- 当前月支出/预算，带进度条

**组织位置卡片：**
- 报告给：可点击的代理名称（链接到其详细信息页面）
- 直接下属：向此代理汇报的代理列表（可点击）

#### 配置选项卡

具有与创建对话框相同部分（适配器、运行时、心跳策略）的可编辑表单，但预先填充了当前值。使用内联编辑——单击值进行编辑，按 Enter 或失焦通过 `agentsApi.update()` 保存。

部分：
- **身份**：名称、标题、角色、报告给、能力
- **适配器配置**：当前适配器类型的所有适配器特定字段
- **心跳策略**：启用/禁用、间隔、唤醒触发器、冷却时间
- **运行时**：上下文模式、预算、超时、宽限期、环境变量、额外参数

每个部分都是一个可折叠卡片。保存是按字段进行的（在模糊/回车时 PATCH），而不是单个表单提交。验证错误内联显示。

#### 运行选项卡

这是主要的活动/历史视图。显示心跳运行的分页列表，最近优先。

**运行列表项：**
```
[StatusIcon] #run-id-short   source: timer     2 min ago     1.2k tokens   $0.03
             "Reviewed 3 PRs and filed 2 issues"
```

每行字段：
- 状态图标（绿色对勾 = 成功，红色 X = 失败，黄色旋转器 = 运行中，灰色时钟 = 已排队，橙色超时 = timed_out，斜杠 = 已取消）
- 运行 ID（短，前8个字符）
- 调用源芯片（timer、assignment、on_demand、automation）
- 相对时间戳
- 令牌使用摘要（总输入 + 输出）
- 成本
- 结果摘要（结果或错误的第一行）

**单击运行**会以内联（手风琴展开）或滑出面板的形式打开运行详细信息，显示：

- 完整状态时间线（queued -> running -> outcome），带时间戳
- 会话前后
- 令牌明细：输入、输出、缓存输入
- 成本明细
- 错误消息和错误代码（如果失败）
- 退出代码和信号（如果适用）

运行详细信息中的**日志查看器**：
- 流式传输运行的 `heartbeat_run_events`，按 `seq` 排序
- 每个事件呈现为带有时间戳、级别（颜色编码）和消息的日志行
- `stdout`/`stderr` 类型的事件以等宽字体显示
- 系统事件以不同的样式显示
- 对于正在运行的运行，通过 WebSocket 事件（`heartbeat.run.event`、`heartbeat.run.log`）自动滚动并附加实时日志
- "查看完整日志"链接从 `heartbeatsApi.log(runId)` 获取并在可滚动的等宽容器中显示
- 截断：默认显示最后200个事件，"加载更多"按钮可获取更早的事件

#### 问题选项卡

保持原样：分配给此代理的问题列表，带状态，可点击导航到问题详细信息。

#### 成本选项卡

扩展现有的成本选项卡：

- **累积总计** 来自 `agent_runtime_state`：总输入令牌、总输出令牌、总缓存令牌、总成本
- **月度预算** 进度条（当前月支出 vs 预算）
- **每运行成本表**：日期、运行 ID、令牌输入/输出/缓存、成本——可按日期或成本排序
- **图表**（扩展）：过去30天每日支出的简单条形图

### 属性面板（右侧边栏）

现有的 `AgentProperties` 面板继续显示快速一览信息。添加：
- 会话 ID（截断，带复制按钮）
- 上次错误（如果有，以红色显示）
- "查看配置"的链接（滚动到/切换到配置选项卡）

## 3. 代理列表页面

### 当前状态

显示代理的平面列表，带状态徽标、名称、角色、标题和预算条。

### 改进

**在标题中添加"新建代理"按钮**（加号图标 + "新建代理"），打开创建对话框。

**添加视图切换**：列表视图（当前）和组织结构图视图。

**组织结构图视图：**
- 显示报告层次结构的树状布局
- 每个节点显示：代理名称、角色、状态徽标
- CEO 在顶部，直接下属在下方，等等
- 使用 `agentsApi.org(companyId)` 端点，该端点已返回 `OrgNode[]`
- 单击节点导航到代理详细信息

**列表视图改进：**
- 在每行添加适配器类型作为小芯片/标签
- 添加"最后活动"相对时间戳
- 如果代理当前有正在运行的心跳，添加运行指示器（动画点）

**筛选：**
- 选项卡筛选：全部、活动、已暂停、错误（类似于问题页面模式）

## 5. API 表面

所有端点都已存在。V1 不需要新的服务器工作。

| 操作 | 端点 | 使用者 |
|--------|----------|---------|
| 列出代理 | `GET /companies/:id/agents` | 列表页面 |
| 获取组织树 | `GET /companies/:id/org` | 组织结构图视图 |
| 创建代理 | `POST /companies/:id/agents` | 创建对话框 |
| 更新代理 | `PATCH /agents/:id` | 配置选项卡 |
| 暂停/恢复/终止 | `POST /agents/:id/{action}` | 标题操作 |
| 重置会话 | `POST /agents/:id/runtime-state/reset-session` | 溢出菜单 |
| 创建 API 密钥 | `POST /agents/:id/keys` | 溢出菜单 |
| 获取运行时状态 | `GET /agents/:id/runtime-state` | 概述选项卡、属性面板 |
| 调用/唤醒 | `POST /agents/:id/heartbeat/invoke` | 标题调用按钮 |
| 列出运行 | `GET /companies/:id/heartbeat-runs?agentId=X` | 运行选项卡 |
| 取消运行 | `POST /heartbeat-runs/:id/cancel` | 运行详细信息 |
| 运行事件 | `GET /heartbeat-runs/:id/events` | 日志查看器 |
| 运行日志 | `GET /heartbeat-runs/:id/log` | 完整日志视图 |

## 6. 实施顺序

1. **新建代理对话框** — 解决从 UI 创建代理的问题
2. **代理列表改进** — 添加新建代理按钮、选项卡筛选、适配器芯片、运行指示器
3. **代理详细信息：配置选项卡** — 可编辑的适配器/心跳/运行时配置
4. **代理详细信息：运行选项卡** — 运行历史列表，带状态、令牌、成本
5. **代理详细信息：运行详细信息 + 日志查看器** — 带流式日志的可扩展运行详细信息
6. **代理详细信息：概述选项卡** — 摘要卡片、组织位置
7. **代理详细信息：成本选项卡** — 扩展的成本明细
8. **组织结构图视图** — 列表页面上的树状可视化
9. **属性面板更新** — 会话 ID、上次错误

步骤1-5是核心。步骤6-9是润色。
