title: 编写技能
summary: SKILL.md 格式和最佳实践
---

技能是智能体可以在其心跳期间调用的可重用指令。它们是教授智能体如何执行特定任务的 markdown 文件。

## 技能结构

技能是一个包含 `SKILL.md` 文件和 YAML 前置内容的目录：

```
skills/
└── my-skill/
    ├── SKILL.md          # 主技能文档
    └── references/       # 可选支持文件
        └── examples.md
```

## SKILL.md 格式

```markdown
---
name: my-skill
description: >
  关于此技能功能和何时使用的简短描述。
  这充当路由逻辑 — 智能体读取此内容以决定
  是否加载完整的技能内容。
---

# 我的技能

智能体的详细说明...
```

### 前置字段

- **name** — 技能的唯一标识符（kebab-case）
- **description** — 路由描述，告诉智能体何时使用此技能。将其编写为决策逻辑，而不是营销文案。

## 技能在运行时如何工作

1. 智能体在其上下文中看到技能元数据（名称 + 描述）
2. 智能体决定该技能是否与其当前任务相关
3. 如果相关，智能体加载完整的 SKILL.md 内容
4. 智能体遵循技能中的说明

这保持了基础提示很小 — 完整的技能内容仅在需要时按需加载。

## 最佳实践

- **将描述编写为路由逻辑** — 包括"何时使用"和"不要使用时"的指导
- **具体且可操作** — 智能体应该能够毫无歧义地遵循技能
- **包含代码示例** — 具体的 API 调用和命令示例比散文更可靠
- **保持技能专注** — 每个技能一个关注点；不要组合不相关的程序
- **谨慎引用文件** — 将支持详细信息放在 `references/` 中，而不是使主 SKILL.md 臃肿

## 技能注入

适配器负责使其智能体运行时可以发现技能。`claude_local` 适配器使用带有符号链接和 `--add-dir` 的临时目录。`codex_local` 适配器使用全局技能目录。有关详细信息，请参阅[创建适配器](/适配器/02%20创建适配器%20creating-an-adapter)指南。
