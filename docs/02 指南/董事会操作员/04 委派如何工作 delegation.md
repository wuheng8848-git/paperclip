---
title: 委派如何工作
summary: CEO如何将目标分解为任务并分配给智能体

委派是 Paperclip 最强大的功能之一。你设定公司目标，CEO 智能体自动将它们分解为任务并分配给合适的智能体。本指南从你作为董事会操作员的角度解释完整的生命周期。

## 委派生命周期

当你创建一个公司目标时，CEO 不只是确认它——它会制定计划并动员团队：

```
你设定一个公司目标
  → CEO 在心跳时醒来
  → CEO 提出策略（为你创建一个批准）
  → 你批准策略
  → CEO 将目标分解为任务并分配给下属
  → 下属醒来（由分配触发的心跳）
  → 下属执行工作并更新任务状态
  → CEO 监控进度、解除阻塞并升级
  → 你在仪表板和活动日志中看到结果
```

每个步骤都是可追踪的。每个任务通过父层次结构链接回目标，因此你总是可以看到为什么正在进行工作。

## 你需要做什么

你的角色是战略监督，而不是任务管理。委派模型期望你做的是：

1. **设定清晰的公司目标。** CEO 基于这些目标工作。具体、可衡量的目标会产生更好的委派。"构建登录页面"是可以的；"在周五前交付带有注册表单的登录页面"更好。

2. **批准 CEO 的策略。** 在审查你的目标后，CEO 向批准队列提交策略提案。审查它，然后批准、拒绝或请求修订。

3. **批准雇佣请求。** 当 CEO 需要更多能力时（例如，前端工程师来构建登录页面），它会提交雇佣请求。你在批准前审查提议智能体的角色、能力和预算。

4. **监控进度。** 使用仪表板和活动日志来跟踪工作流程。检查任务状态、智能体活动和完成率。

5. **仅在停滞时干预。** 如果进度停止，请按顺序检查这些：
   - 你的队列中是否有待处理的批准？
   - 智能体是否已暂停或处于错误状态？
   - CEO 的预算是否已用完（超过80%时，它仅专注于关键任务）？

## CEO 自动做什么

你**不需要**告诉 CEO 让特定的智能体参与。在你批准其策略后，CEO：

- **将目标分解为具体任务**，具有清晰的描述、优先级和验收标准
- **根据角色和能力将任务分配给正确的智能体**（例如，工程任务给 CTO 或工程师，营销任务给 CMO）
- **创建子任务** 当工作需要进一步分解时
- **雇佣新智能体** 当团队没有能力实现目标时，如果启用了公司设置，则雇佣批准可用
- **在每次心跳时监控进度**，检查任务状态并解除下属的阻塞
- **向你升级** 当遇到无法解决的问题时——预算问题、阻塞的批准或战略模糊性

## 常见委派模式

### 扁平层次结构（小团队）

对于只有3-5个智能体的小公司，CEO 直接委派给每个下属：

```
CEO
 ├── CTO         （工程任务）
 ├── CMO         （营销任务）
 └── Designer    （设计任务）
```

CEO 直接分配任务。每个智能体独立工作并汇报状态。

### 三级层次结构（较大团队）

对于较大的组织，经理进一步向下委派：

```
CEO
 ├── CTO
 │    ├── 后端工程师
 │    └── 前端工程师
 └── CMO
      └── 内容作家
```

CEO 将高级任务分配给 CTO 和 CMO。他们将这些任务分解为子任务并分配给自己的下属。你只与 CEO 交互——其余的自动发生。

### 按需雇佣

CEO 可以作为唯一的智能体开始，并根据工作需要雇佣：

1. 你设定一个需要工程工作的目标
2. CEO 提出一个包括雇佣 CTO 的策略
3. 你批准雇佣
4. CEO 将工程任务分配给新的 CTO
5. 随着范围扩大，CTO 可能会请求雇佣工程师

这种模式让你从小处开始，并根据实际工作而不是预先规划来扩展团队。

## 故障排除

### "为什么 CEO 不委派？"

如果你设定了目标但什么也没发生，请检查这些常见原因：

| 检查项 | 查找什么 |
|-------|-----------------|
| **批准队列** | CEO 可能已提交了等待你批准的策略或雇佣请求。这是最常见的原因。 |
| **智能体状态** | 如果所有下属都已暂停、终止或处于错误状态，CEO 没有可以委派的对象。检查智能体页面。 |
| **预算** | 如果 CEO 超过其月度预算的80%，它只专注于关键任务，可能会跳过较低优先级的委派。 |
| **目标** | 如果没有设定公司目标，CEO 就没有工作依据。首先创建一个目标。 |
| **心跳** | CEO 的心跳是否启用并正在运行？检查智能体详细页面以获取最近的心跳历史记录。 |
| **智能体指令** | CEO 的委派行为由其 `AGENTS.md` 指令文件驱动。打开 CEO 智能体的详细页面，验证其指令路径是否已设置，并且文件是否包含委派指令（子任务创建、雇佣、分配）。如果 AGENTS.md 缺失或没有提到委派，CEO 就不知道要分解目标并分配工作。 |

### "我必须告诉 CEO 让工程和营销参与吗？"

**不。** 在你批准其策略后，CEO 会自动委派。它知道组织结构图，并根据每个智能体的角色和能力分配任务。你设定目标并批准计划——CEO 处理任务分解和分配。

### "一个任务似乎卡住了"

如果某个特定任务没有进展：

1. 检查任务的评论线程——分配的智能体可能已经发布了阻塞点
2. 检查任务是否处于 `blocked` 状态——阅读阻塞点评论以了解原因
3. 检查分配智能体的状态——它可能已暂停或超出预算
4. 如果智能体卡住了，你可以重新分配任务或添加带有指导的评论
