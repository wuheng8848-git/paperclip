---
title: 组织结构
summary: 报告层次结构和指挥链
---

Paperclip 强制执行严格的组织层次结构。每个智能体只向一个经理汇报，形成以 CEO 为根的树。

## 工作原理

- **CEO** 没有经理（向董事会/人类操作员汇报）
- 每个其他智能体都有一个 `reportsTo` 字段指向他们的经理
- 创建后，你可以从 **智能体 → 配置 → 报告给**（或通过 `PATCH /api/agents/{id}` 和 `reportsTo`）更改智能体的经理
- 经理可以为他们的下属创建子任务并委派
- 智能体沿指挥链升级阻塞点

## 查看组织结构图

组织结构图在 Web UI 的智能体部分可用。它显示带有智能体状态指示器的完整报告树。

通过 API：

```
GET /api/companies/{companyId}/org
```

## 指挥链

每个智能体都可以访问他们的 `chainOfCommand` —— 从他们的直接报告到 CEO 的经理列表。这用于：

- **升级** — 当智能体被阻塞时，他们可以重新分配给他们的经理
- **委派** — 经理为下属创建子任务
- **可见性** — 经理可以看到下属正在做什么

## 规则

- **无循环** — 组织树严格无环
- **单一父级** — 每个智能体只有一个经理
- **跨团队工作** — 智能体可以接收来自报告线之外的任务，但不能取消它们（必须重新分配给他们的经理）
