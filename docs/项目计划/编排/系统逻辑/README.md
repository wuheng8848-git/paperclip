# 系统逻辑（编排运行真值）

**用途：** 把控制面「代码实际怎么跑」写成可读的编排文档，避免只能靠翻 `heartbeat.ts` 或让 AI 现场翻译。

**读法：先薄后厚**

| 层级 | 路径模式 | 写什么 |
| --- | --- | --- |
| **0 本页** | `系统逻辑/README.md` | 索引：有哪些编排域、各自一句话 |
| **1 域总览（薄）** | `系统逻辑/<编排域>.md` | 产品原则、生命周期简图、子环节目录、与代码模块的对应关系 |
| **2 子环节（厚）** | `系统逻辑/<编排域>/<子环节>.md` | 触发条件、状态机、与相邻环节边界、实现落点、改代码检查清单 |

**与其它目录的关系**

- **`doc/`、`编排/*SPEC*`**：产品契约与验收口径（要什么）。
- **本目录**：运行逻辑与实现指导（现在怎么跑、准备怎么改）。
- **`执行/`、`探查/`、`最佳实践/`**：事务、个案、排障；结论应回写本子环节文档，而不是只在探查里发霉。

**命名约定**

- 域文档用 **业务编排名**（如「事务运行结束编排」），不用执行单编号（如 HB-010）。
- 子环节用 **环节在链路里的角色**（如「关单后运行结案对账」），少用实现函数名当标题。
