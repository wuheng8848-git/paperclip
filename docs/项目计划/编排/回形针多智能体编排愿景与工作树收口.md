# 21 — 回形针多智能体编排愿景与工作树收口（长期）

**定位：** 跨工单的工作方式愿景 —— **探查 → 执行 → 测试 → 工作树 →（可选）第三方审 diff → 合并 PR** —— 在 Paperclip 控制面下如何常态化，而不是单次人肉喊话。

**关系：** 不与 **[05 为何用回形针-编排链路与个人角色](05%20为何用回形针-编排链路与个人角色%202026-05-14.md)**、**[20 Paperclip 认知压缩](20.Paperclip认知压缩-第一轮总结.md)** 重复叙事；本篇只补 **「多智能体 + 工作树 + 收口」** 这一条线的期望。

---

## 1. 愿景陈述（产品意图）

- **控制面（Paperclip）**：工单状态、指派、活动日志、runs **可审计**；董事会派单与租户边界清晰。  
- **执行面（多智能体 / CLI）**：可在 **独立 worktree** 并行试错，避免互相踩未提交树。  
- **收口**：默认路径是 **绿测 / 冒烟证据写回 issue**，再由指定角色 **审 diff**（人或专用评审 agent），最后 **合并 PR**。  
- **技能与上下文**：按角色与阶段挂载技能（参见 **[08 系列](08%20关于技能的选型和配置.md)**），避免「全仓技能灌给每一棒」。
