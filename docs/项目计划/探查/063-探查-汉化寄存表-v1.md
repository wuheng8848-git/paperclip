## 7. 执行策略：Flash 探查 vs 强模型改码

### 7.1 可派给 Flash（DeepSeek v4 Flash）的执行探查/简单替换

这些任务的特点是：**同质重复、无逻辑变更、纯字符串替换 + i18n 键追加**。

| 批次 | 事务标题 | 原因 |
| --- | --- | --- |
| P0-A | 事务主路径导航与状态标签 | 大量 PropertyRow label 是纯字符串替换；i18n.ts 已有分组模式可复用 |
| P1-A | 事务主路径按钮角色标签与错误兜底 | 同质替换，follow 已有 i18n 引用模式 |
| P2-A | 事务主路径占位符错误消息与文档冲突处理 | 量大但同质，纯 UI 静态替换 |
| P2-B | PluginSettings 配置表单 | 组件内硬编码英文，纯字符串替换 |
| P2-C | CompanyInvites/CompanySkills 工具函数 | 少量函数内英文，量小简单 |
| P3-A | 事务主路径 tooltip 与低频状态 | 收尾工作，量小简单 |

### 7.2 需强模型（优先用 Claude / Qwen 强模型）

这些任务的特点是：**涉及服务端代码、模板参数、enum/value 映射、需理解跨层依赖**。

| 批次 | 事务标题 | 原因 |
| --- | --- | --- |
| P0-B | CompanyExport/CompanyImport Toast 与按钮 | 需新建 `companyExportPage` / `companyImportPage` i18n 对象；模板字符串 `${n}` 需函数形式 |
| P0-C | Routines/RoutineDetail 页面 | 需新建 `routinesPage` / `routineDetailPage` i18n 对象；涉及状态枚举映射 |
| P1-B | CompanyExport/CompanyImport 完整页面 | 两个页面完全无 i18n 基础设施，需从零建立 + 字段组件联动 |
| P1-C | WorkspaceOperationStatusBadge | 涉及组件逻辑（CSS capitalize → 中文化映射函数），不是纯字符串替换 |
| P1-D | 服务端滞留恢复评论 | 改 `server/src/services/recovery/service.ts`，理解 recovery 上下文 |
| P2-D | 运行观测域 env fallback | `formatEnvForDisplay` 函数逻辑变更 |
| P2-E | 服务端 recovery 模块英文评论 | 改服务端代码，需理解 recovery 生命周期 |
| P2-F | 例行任务历史与变量编辑 | 涉及模板字符串、枚举状态映射、`formatInviteState()` 等 |
| P3-B | 转写预览标签 | 需重构 `transcript-preview.ts` 引用现有 i18n 映射 |
| P3-C | 服务端技术元数据 | 改服务端代码 |
