# 探查（本目录说明）

本目录收 **个案探查、长表、取证结论**：偏「**发生了什么、为什么、证据在哪**」。可复用的操作路径与命令优先收口到 **[`../最佳实践/`](../最佳实践/README.md)**；要落地改代码、验收闭环的工单在 **[`../执行/`](../执行/)**。

## 命名与取号

- 专文文件名：**`NNN-中文主题.md`**（三位序号 **`001` 起**，与 **`执行/`**、**`最佳实践/`** **不同号池**）。  
- **`NNN` 的含义**、Git/本地时间排序、改名时要修的链接类型：见 **[`../README.md`](../README.md)** 专节 **「探查」目录：文件名编号与生成顺序**。  
- **`README.md`**（本文件）**不加序号**，仅供目录说明；**不作**排序里的「001」占位。

## 写单后必做

新开篇、更名或删除专文时，须 **同一轮** 更新 **[`../index.md`](../index.md)** **「### 探查」** 段（按 **`001`→递增**），并修补 **`docs/`** 内指向旧文件名的链接。总则见 **[`../README.md`](../README.md)** **「全量索引维护」**。
