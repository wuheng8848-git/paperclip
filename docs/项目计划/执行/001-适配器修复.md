## 1. 需求

### 要什么

Paperclip 与部分**外部适配器**组合下，无法在控制面统一管理/挂载技能，智能体「技能」区域提示须在适配器中自行管理；适配器链路本身也可能存在缺陷。结果是：**以技能为枢纽的编排**（串联智能体 · 公司 · 项目 · 事务）难以稳定开展。

需同时区分两层问题：**智能体代配智能体时的任务/向导是否缺约束**，与 **适配器运行时是否不可靠**。

### 验收直觉

1. 能明确卡点落在 **适配器能力** / **Paperclip UI·API** / **操作流程与文档**中的哪一类或组合。
2. 对当前主攻适配器有一条可复述、可重复的 **正确配置路径**（步骤或脚本级）。
3. 有一条 **最小工单**可走通：`创建或编辑 agent →（若平台支持）挂技能 → 验证适配器 CLI 侧行为**符合预期。
