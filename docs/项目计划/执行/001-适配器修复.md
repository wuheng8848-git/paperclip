## 1. 需求

### 要什么

Paperclip 与部分**外部适配器**组合下，无法在控制面统一管理/挂载技能，智能体「技能」区域提示须在适配器中自行管理；适配器链路本身也可能存在缺陷。结果是：**以技能为枢纽的编排**（串联智能体 · 公司 · 项目 · 事务）难以稳定开展。

需同时区分两层问题：**智能体代配智能体时的任务/向导是否缺约束**，与 **适配器运行时是否不可靠**。

**本单不再以「最小事务 / issue→run→done 首次走通」为主验收：** 该链路已在实例上跑通；归档与排障入口见 **§1.4**。当前主任务是：**对照地图收口剩余适配器缺陷与文档索引**，让人能一眼找到「某适配器」的权威配置/排障步骤，而不是从零摸索。

### 验收直觉

1. 能明确卡点落在 **适配器能力** / **Paperclip UI·API** / **操作流程与文档**中的哪一类或组合。
2. **可重复配置路径**：以 **§1.4 文档地图**为准——地图中列出的执行单 / 探查 / 最佳实践是否覆盖当前主攻适配器；**缺口**（缺专文、缺索引、与实例不一致）记入 **§3** 并决定是补文、补表还是改代码。
3. ~~最小事务走通~~（**已达成**，不作为本单重复目标）。
