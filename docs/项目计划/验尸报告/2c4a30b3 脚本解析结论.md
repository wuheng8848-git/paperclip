## 3. 乱码对照（关键证据）

### 3.1 Paperclip 落库（`adapter.invoke.prompt`）— 部分正常

| 片段 | 在 invoke 中 |
|------|----------------|
| `## Paperclip 唤醒负载` | **正常** |
| `ROU-3 测试闭环-验证cursor适配器-列出当前目录文件` | **正常** |
| Continuation Summary 内 `测试闭环-验证cursor…` | **已乱**（`` 替换符 / 错码） |
| Agent 名 `计划` | 摘要里为 **`ƻ (cursor)`** |

说明：**不是 Paperclip 拼 wake 模板时整段坏了**，而是 **retry 继承的 `continuationSummary` 在上轮已脏**（路径 A：失败 run 摘要生成时 stdout 解码问题，见 021）。

### 3.2 run-log（Cursor stream-json 回显的 user 消息）— 整段乱

ndjson 第一条 **user** 行（stdin 镜像）开头为：

```text
## Paperclip 鍞ら啋璐負荷…
```

（应为「唤醒负载」。）

| 位置 | 唤醒标题 | 含义 |
|------|----------|------|
| `adapter.invoke` | 唤醒负载 ✓ | Paperclip 写入事件时仍对 |
| run-log `user` | 鍞ら啋… ✗ | **进 Cursor 子进程后 / CLI 回显 stdout 时变坏** |

与 CodeBuddy `8956f084` 的差异：CodeBuddy 乱在 **tool_result**；Cursor 本轮乱在 **整包 stdin 回显**。
