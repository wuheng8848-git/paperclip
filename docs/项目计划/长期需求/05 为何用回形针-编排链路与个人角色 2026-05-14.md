### 0.2 五仓是什么（工程 / 资料目录级）

**记忆顺序（你口述）：** 贾维斯 → Token → 回形针 → 工具优化 → Routic。

| 仓 | 典型位置（本机习惯写法） | 真值 / 职责（人话） |
| --- | --- | --- |
| **1 · Jarvis** | `code/jarvis`（等） | **个人主控**：时间、注意力、跨域索引、记忆、今日主线裁决 |
| **2 · Token** | `token-bridge-v2` | **工程真源**：产品代码、里程碑、任务单、部署与验收证据 |
| **3 · 回形针** | `code/paperclip-latest-20260512` 等 | **执行平面**：公司 / 智能体 / 工单 / 心跳 / run ledger / 预算与审批；**令工人可观察地干活** |
| **4 · 工具优化** | `工具优化` | **方法论真源**：AI 工具链、模型分工、编排认知、规则、skills |
| **5 · Routic** | `routic` | **经营资料真源**：公司主体、供应商、市场、运营、财务、客户、业务策略 |

**仍不算独立一仓的：** 各 **CLI**（Cursor / CodeBuddy / Codex …）是 **回形针调起的运行时工人**，长期真值不进 CLI 独占仓；**Hermes 适配器插件**是回形针**内部组件类型**，不是第六仓（见 §0.4）。

### 0.3 两个「Routic」不要混

| 叫 Routic 的 | 是什么 |
| --- | --- |
| **五仓之 `routic` 目录** | **经营域资料原件** |
| **Paperclip 里 slug `routic` 的公司** | 控制平面上的**本机蜂群 / 组织模板**；**≠** 经营资料全集 |

未加限定时，**「routic 经营」**优先指 **五仓 · Routic 资料**；谈 Board / agent / issue 时，写明 **Paperclip 租户**。

### 0.4 「赫尔墨斯」与「Hermes 适配器」

- **Hermes（适配器）**：本 fork 常见用法 = Paperclip **外置适配器插件**，属 **回形针仓内部件**，不是与五仓并列的第六仓。  
- **赫尔墨斯（信使 / 手机 / 发布通道）**：若在实现上单列，宜归 **Jarvis 延伸或独立集成文书**；与 Paperclip API 的对接以 `01.项目需求说明.md` 开放问题与实装为准，本文**不写死唯一链路**。

| 块 | 含义（人话） | 五仓 |
| --- | --- | --- |
| **Jarvis** | 个人主控、记忆与主线裁决 | **贾维斯** |

## 3. 回形针在链路里的位置：指令怎么流

**结构**兼顾五仓并列与执行纵深：

```text
吴衡
    → Jarvis（裁决与汇总）
        ├→ Token / 工具优化 / Routic 资料（各仓真源读写边界各自守）
        └→ Paperclip（回形针：派工与工人账本）
            → CLI 工人
            → HeartbeatRun / ResultJson / 活动与 recovery
            → 摘要可回流 Jarvis（非默认啃全量 log）
```

- **今日做什么、停什么** —— **Jarvis**。  
- **任务如何拆给哪几个 AI 工人、如何留证** —— **Paperclip**。  
- **工程是否交付、经营判断** —— 分别以 **Token / Routic 资料** 为真源；Paperclip **跑完单 ≠ 自动回填** 各仓原件。

## 4. 「回形针正常之后」要帮我干哪些活

1. **工人账本**：谁跑、跑什么、成本与结果、recovery——**≠ 五仓其它资料已自动更新**。  
2. **可重复派工**：issue → heartbeat → run → 收尾。  
3. **门禁与预算**：审批、硬停、公司边界（见实现规格）。  
4. **稳定唤起 CLI**：适配器可换，生命周期与证据可测。  
5. **API**：供 Jarvis / 脚本拉 **摘要**（与 Jarvis `四仓定位.md` §8、及本仓库 [`06`](06%20五仓定位-回形针侧口径%202026-05-15.md) §8 同旨）。

---

## 5. 刻意不做什么（防止目标漂移）

- **不把回形针当成 Jarvis**（个人主控仍是一仓独大）。  
- **不把回形针当成 Routic 资料库**（经营原件仍在 **Routic 仓**）。  
- **不把「成为容器专家」当成功标准**——除非当下阻塞迭代。  
- **不把巨型手写配置表当常态**——见姊妹篇。  
- **不把 CLI 穷举当主业**。
