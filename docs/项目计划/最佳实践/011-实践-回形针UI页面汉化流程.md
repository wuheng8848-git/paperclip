# 实践：回形针 UI 单页/单模块汉化流程

本仓库 UI **不设运行时语言切换**：字典集中在 **`ui/src/lib/i18n.ts`**（大型 **`const` 对象**，纯 zh-CN）。下列步骤摘自 **`029` / `030` / `031`** 三份执行单的共性做法，用于任意「还要看板的某一页」——先做一页再在事务里留证据即可。

**溯源事务：** [029 设置页全量汉化批量执行](../执行/029-设置页全量汉化批量执行.md)、[030 IssueProperties 属性面板](../执行/030-IssueProperties右侧属性面板汉化.md)、[031 Activity 活动页](../执行/031-Activity活动页汉化.md)。

**菜单/页面叫什么（写文档、执行单、助手步骤）：** 以 **[023 Board 界面中文口径](023-实践-Board界面中文口径与文档对照表.md)** 与 `i18n.ts` 为准（例如 **运行清单**，勿写「编排注入」）。

---

## 1. 前置确认

| 项 | 说明 |
| --- | --- |
| 字典真源 | `ui/src/lib/i18n.ts`：新增键时 **按页面/模块分组**（如 `xxxPage`、`xxxLabels`），一般 **在文件末尾追加** const，避免拆散既有结构时多看一眼现有分组命名。 |
| 无 i18next | 不要用「搜不到 `useTranslation` 就当没有 i18n」；以 **`i18n.ts` + 组件 `import { … } from "../lib/i18n"`** 为准。 |
| 设计/token | 组件样式与可读性对齐仓库 **design-guide** 约定；内置一览路由 **`/design-guide`**（技能入口路径随宿主配置而异，可在工作区搜 `design-guide`）。 |

---

## 2. 划定范围（译什么、不译什么）

**译：** 用户在当前路由/弹层内 **肉眼可见** 的静态英文标签、按钮、表头、空状态、面包屑、筛选器文案等。

**不译（与三单一致）：**

| 类型 | 示例 |
| --- | --- |
| 技术占位 / 示例配置 | `OPENAI_API_KEY`、示例路径、包名 |
| 运行时动态串 | API 返回名、健康详情、未做映射的枚举原文 |
| Agent / 机器侧文案 | 例如给模板或 CLI 用的 snippet（除非产品明确要求上屏） |
| 标识与导航地址 | `data-testid`、CSS 类名、URL |
| 可选保留 | 国际通用单位（MiB、PID 等）——按产品口径 |

## 3. 推荐操作顺序

### 第一步：盘点英文字符串

- 锁定 **页面入口**（如 `ui/src/pages/*.tsx`）与 **专用组件**（如 `ui/src/components/IssueProperties.tsx`）。
- 必要时连带 **格式化/词典模块**（如 `activity-format.ts`）：活动类文案常在 **展示层 + lib** 两处。
- 用检索或结构化扫描列出 **file → 行号/片段 → 文案**，避免大文件盲替换。

### 第二步：在 `i18n.ts` 增加分组对象

- **一类页面一块 const**（例如 `issuePropertiesPage`、`activityPage`），其下 **camelCase 键**；复杂列表再拆 `xxxLabels`、`xxxVerbs`。
- **模板句**写成 **函数**（如 `description(name: string)`），不要在 TSX 里拼接中英混杂裸字符串。
- **枚举 / value→label**（见下节）：**value 保持 API 约定**，只汉化 **展示用 label**。

### 第三步：改 TSX / TS（仅换文案）

- 组件顶部：`import { fooPage } from "../lib/i18n";`（路径按层级调整为 `../lib` 或 `../../lib`）。
- **替换硬编码字符串 → `fooPage.key`**；**不改** 业务逻辑、类型、请求参数名、CSS、布局（除非事务另有要求）。
- **超大组件**（千余行）：优先 **小块 edit** 或分文件段落，避免脚本误替换。

### 第四步：验证

- 仓库根或 `ui/` 下执行类型检查，例如：

```bash
npx tsc --noEmit
```

（Windows 下可用 `findstr` 过滤只看 error；已知与本工作无关的 **tsconfig 引用噪声** 可在事务里注明，避免反复纠结。）

## 4. 高频踩坑（030 / 031 浓缩）

| 情况 | 做法 |
| --- | --- |
| **下拉 / 选项**：常量映射 `VALUE → Label` | **保留 value** 供 API；另建 **仅展示用** 的中文 label 表或 `i18n` 函数，禁止把 value 改成中文导致请求失败。 |
| **`forAdapter()` 等返回「给人看的数组」** | 分清 **提交用 value** 与 **展示用 label**；必要时 **双向映射**（label→value）仅在交互边界使用。 |
| **活动/日志类动词、状态补丁** | 除组件外，检查 **`ui/src/lib/*format*.ts`** 或同类 **字典对象**，避免遗漏「结构化渲染」路径。 |
| **误判没有 i18n** | 搜 **`i18n.ts`**、搜 **`from "../lib/i18n"`** 或字典里已有 **中文 **Unicode** 片段**，比搜 `i18next` 可靠。 |

---

## 5. 并行与体量（029 经验）

- **多文件、同质替换** 时可拆并行任务，但 **同一文件** 建议单人串行或锁文件，避免冲突。
- **字典键** 宁可在 CR 时 **多核对一页预览**，也不要漏键导致运行时 `undefined` 上屏。

---

## 6. 收尾（协作）

- **执行单**里写清：触及文件列表、新增键分组名、验证命令输出要点。
- **若形成可重复 playbook**：维持本文；若只是一次性边角文案，可口头链回 **`i18n.ts`** 不必扩写本文。

## 7. 服务端 / 适配器里的中文句（不在 `ui/`）

**看板以外的**拼在 **`server/`**、**`packages/adapters/`** 里、打 **`[paperclip]`** 或回给 API 的人机句：**不经过** `i18n.ts`，改法见 **[`019-实践-服务端与适配器人机文案汉化.md`](019-实践-服务端与适配器人机文案汉化.md)**（人类只贴「原句 → 目标中文」）。
