本目录收 **可复用的打法**：偏「**下次照哪条路做、命令与顺序是什么**」。个案里的 **证据链、归因表、长结论** 仍在 **[`../探查/`](../探查/README.md)**；要 **改代码、登 `00`、验收闭环** 的在 **[`../执行/`](../执行/)**。

## 命名与取号

- 专文文件名：**`NNN-中文主题.md`**（三位序号 **`001` 起**，与 **`执行/`**、**`探查/`** **不同号池**）。  
- **`NNN` 的含义**、Git/本地时间排序、改名时要修的链接类型：见 **[`../README.md`](../README.md)** 专节 **「最佳实践」目录：文件名编号与生成顺序**。  
- **`README.md`**（本文件）**不加序号**，仅供目录说明；**不作**排序里的「001」占位。

## 写单后必做

新开篇、更名或删除专文时，须 **同一轮** 更新 **[`../index.md`](../index.md)** **「### 最佳实践」** 段（**`README.md` 置顶**，其余按 **`001`→递增**），并修补 **`docs/`** 内指向旧文件名的链接。总则见 **[`../README.md`](../README.md)** **「全量索引维护」**。
