# 019-实践-服务端与适配器人机文案汉化

**给人一条规矩、给助手一条默认：** 人类只贴**对照文案**，助手按本节改代码，**不**再追问「怎么汉化后端」。

---

## 1. 人类怎么发（唯一口令）

一条消息里写清两类字符串即可：

- **现文案**（仓库里已有的整句，英/中均可）  
- **目标中文**（最终给人看的）

不必写长背景；**一句一对**最省事。

---

## 2. 分层（和前端对照）

| 落点 | 改哪里 | 详文 |
| --- | --- | --- |
| **看板 / UI** | `ui/src/lib/i18n.ts` + 组件引用 | **[`011-实践-回形针UI页面汉化流程.md`](011-实践-回形针UI页面汉化流程.md)** |
| **API 错误句、服务端拼给人看的说明** | `server/**/*.ts` 命中处 | 本节 §3 |
| **适配器 stdout/stderr、`[paperclip]` 行、注入提示** | `packages/adapters/**`、`packages/adapter-utils/**` 等命中处 | 本节 §3 |

本 fork **默认静态 zh-CN**：**不做**运行时多语言开关，另有工单再说。

---

## 3. 服务端与适配器「通用改法」（默认动作）

1. **全仓搜现文案**（`rg`），覆盖 **`server/`**、**`packages/`**（含适配器），**所有命中一并改掉**，避免一句两文。  
2. **改法**：在命中处把**模板字符串 / 字面量**直接换成**目标中文**；同句重复 ≥2 次可**抽到**同文件顶部常量或 `packages/shared` 可读常量（**不**为一句话上大套 i18n 框架）。  
3. **仍会进 UI 的 API `message`**：与 **`i18n.ts`** 若已有展示映射，**二选一真值**——以**当轮人类贴的文案**为准改掉一端，并**同一轮**消除重复表述。  
4. **不要翻译**：协议字段名、稳定 `error` code、第三方 CLI **原样**输出、测试里**刻意**断言的英文常量（改文案则**同步改测试期望**）。  
5. **自检**：`pnpm -r typecheck`，或最小 **`pnpm exec tsc --noEmit -p server`** + 涉及包。

---

## 4. 明确不做的

- 不把 **[paperclip] 运行日志**再抄进 UI 字典搞「双份」；要旁白另开产品单。  
- 不因汉化改 **HTTP 契约字段**（只改 **值** 的 natural language 部分）。

---

## 5. 人类与助手的分工

| 谁 | 做什么 |
| --- | --- |
| **人类** | 贴 **现句 → 目标句** |
| **助手** | 搜全仓、改、跑 typecheck、**不啰嗦**可改清单 |
