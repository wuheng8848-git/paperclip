# 实践：排障——run 段逻辑（高 token / 多轮 / 工具大 JSON）

本文是 **排障流水线里「针对单次 heartbeat run」的固定顺序**，**总目录见 [020](020-实践-排障指南.md)**；与 **「事务→runs 列表」取证**（[002](002-实践-事务运行记录API取证路径.md)）、**僵尸 run / 唤醒**（[004](004-实践-事务心跳与僵尸run排障.md)）、**外部工具五步**（[012](012-实践-回形针外部开发工具与排障流水线.md)）、**CLI 计量口径**（[015](015-实践-CLI累计计量与成本控制面台账口径.md)）、**无头提示词叠层与三尺子**（[022](022-实践-无头编排提示词叠层与可观测性.md)）互补：**004 偏调度与健康；本文偏「这一条 run 为什么在模型侧又贵又长」**；**022 解释 `promptChars` / 日志 / `input_tokens` 为何对不上**。

---

## 1）先建立心智模型（避免误伤）

| 概念 | 人话 |
| --- | --- |
| **一个 run** | Paperclip 控制面记账的 **一趟执行**（有 `runId`、状态、用量、日志）。 |
| **一个进程** | 常见路径里 ≈ 这趟执行拉起的主 CLI 子进程；**≠** 与 run 数学等价，但日常可这么理解。 |
| **很多「轮」** | 适配器会话里 **模型 ↔ 工具** 反复交替，`resultJson.num_turns` 往往很大——**不是**调度给你派了两百次心搏，而是 **这一趟里面搓了很多圈**。 |
| **input token 很大** | 常与 **圈数多**、**每圈带进上下文的大段 JSON（工具回包）**、**会话累计** 同现；`usageJson` 里也可能含 **prompt cache 命中** 的统计口径，**不等于**全是「按全价新喂了两百万字」。 |

---

## 2）run 段推荐顺序（落地就按步查）

| 步 | 做什么 | 判断什么 |
| --- | --- | --- |
| ① | `GET /api/heartbeat-runs/{runId}` | **状态**、**错误/取消**、`wakeReason`、`exitCode`、`issueCommentStatus`、`livenessState`：是 **控制面掐断** 还是 **跑完**。 |
| ② | 同上读 **`usageJson`**、**`resultJson.num_turns`**（及 `logBytes`/`lastOutputBytes` 量级） | **圈数**是否离谱；用量是否与 **多轮+肥日志** 一致。 |
| ③ | 看板 **`/{公司前缀}/runs/{runId}`**（运行清单：finalPrompt / input / 事件） | **首包就巨胖**，还是 **工具把大 JSON 反复灌进上下文**；必要时对照 **`adapter.invoke`** 载荷。 |
| ④ | **运行详情**里的 transcript / stdout 摘要 | 是否 **重复 `GET` 全量评论**、同一接口打很多次。 |
| ⑤ | 再拆：**唤醒段 / heartbeat-context 已够吃** 仍全库拉 → 偏 **技能与提示**；API 只能粗粒度拉全量 → 偏 **产品与接口形态**。 |

---

## 3）定性口诀

- **取消 + 控制面说明** → 先看 **暂停 / 调度 / 预算 / 掐 run**，别先骂模型胡写。
- **成功但 token 炸** → 先 **②③**（圈数 + 运行清单），再谈改提示还是改取证方式。
- **行为错但 token 正常** → 重点 **`contextSnapshot` / wake**，不是计费。

---

## 4）与「编排闸门」页的关系

看板路由 **`/{公司前缀}/orchestration-gates`** 说明 **服务端调度闸门**（定时、避退、回收等），**不**解释单次 run 里模型喂了多少字。单次 **prompt 拼装** 请用 **运行清单** **`/{公司前缀}/runs/{runId}`**（上表第 ③ 步）。

## 5）改进方向（排障结论落地时）

- **产品/API**：更细的查询参数（分页、`after`、少字段）、控制面 **多下发文摘**（与 **[045](../../探查/025-探查-045-Skill懒加载与上下文封顶-现状与落点.md)** 等「上下文封顶」叙述同向）。
- **技能/提示**：写清 **最小必要取证**，禁止无脑全量拉评论。
- **运维**：本条与 **015** 同看时可区分 **上游 CLI 累计** 与 **本条 run 账面**。
