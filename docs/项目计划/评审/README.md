本目录存放 **[`016-实践-042至053编排开发收口套路`](../最佳实践/016-实践-042至053编排开发收口套路.md)** 中 **③ 评审** 的留痕稿：**开口选项**、人类拍板结论、**无开口**声明等。与 **[`技术设计/`](../技术设计/README.md)** 分列——设计写「准备怎么改」，评审写「分叉定了没有、定哪边」。

## 评审稿怎么写（体例）

对齐 **`26 A1评论唤起档位-评审稿`** 一条路即可，常见块：

1. **标题：** `… — 评审稿（只管拍板）`  
2. **背景一句** — 非技术细节，说清 **为啥现在要人类选**  
3. **`## 本轮已表态`** — 总表 `条 / 结论 / 备注`（已定案就填实词；未定标 **待定**）  
4. **「第 N 条白话」** — 每个大分叉用 **人话** 写 **A/B 选项、好坏、风险**（像 `26` 里 **第 4 条 / 第 5 条白话**）  
5. **分条编号 `## 1. … ## 2. …`** — 一句 **你选哪边** + （略）或摘要  
6. **`## 结案表`** — 勾选式对齐技术设计 / 执行单；写 **评审状态**、**跟进**  
7. **`附`** — 链 **技术设计、探查、母本**；长周期 A1 类也可用 **`26`/`25` 编号文件名**（与执行单 `045-` 前缀可以并存，以人类好搜为准）

**命名：** 执行单驱动事务建议 **`045-…评审稿 日期.md`**；专题/A1 类可与长期需求 **`25`/`26` 同编号**，便于和 **`长期需求/`** 对读（见下表 **26**）。
