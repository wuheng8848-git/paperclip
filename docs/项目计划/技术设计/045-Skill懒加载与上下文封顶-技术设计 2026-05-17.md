## 实现原则（与执行单一致）

1. **命名与复用**：具体字段名以满足契约为准；**优先复用**现有类型与配置位，避免重复建模。
2. **增量修改**：小步扩展、可回滚；拒绝无关大重构。
3. **上游可合并**：保持与上游仓库**可 diff、可吸收**的形态，不因本单把 `server/`、`packages/` 等核心树改到难以 cherry-pick / merge。

## 本单范围（已定案）

- **适配器侧**：仅 **`cursor-local`**、**`codebuddy-local`**、**`qwen-local`** 三包；不在本单修改的适配器不得为「对齐行为」而被动大改。
- **`server/`、`packages/adapter-utils`**：允许**最小增量**（配置透传、共享 helper、与 B1–B3 直接相关的分支），遵守执行单 **「改代码原则」**。

以下 B1–B3 的技术开口在遵守上述原则前提下取舍。

## B1 — Skill 懒加载（默认不把整条 `paperclip` SKILL.md 当首帧上下文）

**思路 A（优先，偏控制面）：** 在 **`comment wake` / 低档位** 路径（可与 `contextSnapshot.commentWakeTier`、041 档位枚举联动）：

- 不把完整 `paperclip` 技能注入 **prompt 前缀**；改为注入 **结构化摘要**（例如：`SKILL.md` 一级标题列表 + 固定字数摘要 + 「按需用工具读 `.paperclip/skills/.../SKILL.md`」指引）。
- **磁盘/ symlink**：仍可保留 `ensure*SkillsInjected` 的全目录供 CLI 按需读；**省的是模型首包 token**，不是删仓库文件。

**思路 B（补充）：** 在 `listRuntimeSkillEntries` 或心跳合并 config 时，为 **非 required** 技能增加 `lazy: true` 元数据，由适配器在 **会话 preamble** 里只列清单不展开正文（需动 `adapter-utils` + **本单限定之三适配器**验证）。

**依赖 041：** 若已有「禁用全 Skill / 仅保留 paperclip 摘要」档位，045 验收可与该档位 **E2E 对齐**。

## B2 — `fallbackFetchNeeded: false` 时去重

- **服务端**：在 `renderPaperclipWakePrompt` 已重复「勿拉 API」的前提下，检查 **instructions / injected skill 摘要** 是否与 wake 块 **语义重复**（例如 execution 契约长段）；**摘要保持一份**（优先保留 wake 渲染层，或抽取共享短链）。
- **适配器**：当 **`fallbackFetchNeeded === false`** 且 **reason === `issue_commented`**（或再收窄），对 **SKILL.md 注入式 preamble** 降格为「指针 + 1 段摘要」；`true` 时维持现状或放宽。

（实现前在单测里用 **normalize 后的 wake payload** fixture 对照 prompt 字数。）

---

## B3 — AGENTS.md / codebuddyMd 封顶

- 在 **评论唤起心跳** 路径定义 **`MAX_AGENT_INSTRUCTIONS_CHARS_COMMENT_WAKE`**（或按比例：与 `renderPaperclipWakePrompt` 输出之和封顶）。
- **折叠策略（草案）**：优先截断 **与当前 `issueId`/ wake 无关** 的章节（需粗粒度 Markdown 分段或约定 `##` 边界）；不足再截尾部。
- **Codex**：文档已提示会额外发现仓库 AGENTS——045 可只做 **Paperclip 注入段** 封顶 + 记录观测；仓库侧 AGENTS **过度注入** 如需一起管，单列后续事务（046+）以免本单爆炸。
