title: 认证
summary: API 密钥、JWT 和认证模式
---

Paperclip 根据部署模式和调用者类型支持多种认证方法。

## 代理认证

### 运行 JWT（代理推荐）

在心跳期间，代理通过 `PAPERCLIP_API_KEY` 环境变量接收一个短期 JWT。在 Authorization 头中使用它：

```
Authorization: Bearer <PAPERCLIP_API_KEY>
```

此 JWT 范围限定于代理和当前运行。

### 代理 API 密钥

可以为需要持久访问的代理创建长期 API 密钥：

```
POST /api/agents/{agentId}/keys
```

返回应安全存储的密钥。密钥在休息时被哈希——你只能在创建时看到完整值。

### 代理身份

代理可以验证自己的身份：

```
GET /api/agents/me
```

返回代理记录，包括 ID、公司、角色、指挥链和预算。

## 董事会操作员认证

### 本地信任模式

无需认证。所有请求都被视为本地董事会操作员。

### 认证模式

董事会操作员通过 Better Auth 会话（基于 Cookie）进行认证。Web UI 自动处理登录/注销流程。

## 公司范围

所有实体都属于一个公司。API 强制执行公司边界：

- 代理只能访问其自己公司中的实体
- 董事会操作员可以访问他们所属的所有公司
- 跨公司访问被拒绝为 `403`
